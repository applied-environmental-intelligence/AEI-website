# Website Update Guide

This guide explains how to update and maintain the AEI website.

## 🎯 Golden Rule

**Always edit files in `src/`, never in `dist/`!**

The `dist/` directory is automatically generated by the build process and any manual edits will be overwritten.

## 📝 Common Updates

### Update Text or Content

1. Open [src/index.html](../src/index.html)
2. Make your changes to the HTML content
3. Save the file
4. Rebuild the site:
   ```bash
   python build.py
   ```
5. Preview locally:
   ```bash
   python -m http.server --directory dist 8000
   ```
6. Commit and push to deploy:
   ```bash
   git add src/index.html
   git commit -m "Update page content"
   git push origin main
   ```

### Add a New Section

1. Edit [src/index.html](../src/index.html)
2. Add your new section HTML where appropriate:
   ```html
   <section id="my-new-section" class="my-section">
     <h2>New Section Title</h2>
     <p>Content goes here...</p>
   </section>
   ```
3. Add corresponding CSS in the `<style>` tag
4. Update navigation links if needed:
   ```html
   <a href="#my-new-section">My Section</a>
   ```
5. Rebuild and test locally
6. Commit and push

### Add a New Demo/Project Page

1. Create a new HTML file in `src/`:
   ```bash
   touch src/project-name.html
   ```
2. Copy the structure from `src/index.html` (header, footer, styles)
3. Add your project content
4. Link to it from the main page:
   ```html
   <a href="project-name.html">View Project</a>
   ```
5. Update `build.py` if needed to ensure the new file is copied:
   - The current build script copies `index.html` explicitly
   - For additional pages, add them to the `copy_static_assets()` function
6. Rebuild and test
7. Commit and push

### Embed YouTube Videos

Add a responsive YouTube embed in `src/index.html`:

```html
<div class="video-container">
  <iframe 
    width="560" 
    height="315" 
    src="https://www.youtube.com/embed/VIDEO_ID_HERE" 
    title="YouTube video player" 
    frameborder="0" 
    allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture" 
    allowfullscreen>
  </iframe>
</div>
```

Add this CSS to make it responsive:

```css
.video-container {
  position: relative;
  padding-bottom: 56.25%; /* 16:9 aspect ratio */
  height: 0;
  overflow: hidden;
  max-width: 100%;
}

.video-container iframe {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
}
```

Replace `VIDEO_ID_HERE` with your YouTube video ID (from the URL after `v=`).

### Add Static Images

1. Create an images directory in src (if it doesn't exist):
   ```bash
   mkdir -p src/assets/images
   ```
2. Add your images to `src/assets/images/`
3. Reference them in HTML with relative paths:
   ```html
   <img src="assets/images/my-photo.jpg" alt="Description">
   ```
4. Update `build.py` to copy the images directory (already handles this)
5. Rebuild and commit

### Update Team Members

1. Edit [src/index.html](../src/index.html)
2. Find the team section (around line 551)
3. Modify existing team member cards or add new ones:
   ```html
   <div class="team-member">
     <img src="URL_TO_PHOTO" alt="Name" />
     <h3>Full Name</h3>
     <p class="title">Job Title</p>
     <p class="bio">
       Biography text here...
     </p>
   </div>
   ```
4. Rebuild and deploy

### Update Contact Information

1. Edit [src/index.html](../src/index.html)
2. Find the contact section (around line 599)
3. The form uses FormSubmit.co - update the email if needed:
   ```html
   <form action="https://formsubmit.co/YOUR_EMAIL" method="POST">
   ```
4. Rebuild and deploy

## 🎬 Regenerate Animations

### When to Regenerate

- Animation parameters changed (colors, timing, resolution)
- Source image updated (`subsurface.png`)
- Bug fixes in animation scripts

### How to Regenerate

The build script automatically regenerates animations. If you want to regenerate them manually:

```bash
# Cable animation only
python scripts/generate_cable_animation.py --output dist/animations/cable_animation.gif

# Subsurface reveal only (requires source image)
python scripts/generate_reveal_subsurface.py \
  --input src/assets/source/subsurface.png \
  --output dist/animations/subsurface_reveal.gif
```

### Modify Animation Parameters

Edit the Python scripts in `scripts/`:

**Cable Animation** (`scripts/generate_cable_animation.py`):
- **Duration**: Change `duration_s` variable (line 44)
- **Frame rate**: Change `fps` variable (line 43)
- **Resolution**: Change `W, H` variables (line 40)
- **Colors**: Change `ink` and `bg_level` variables (lines 170-171)

**Subsurface Reveal** (`scripts/generate_reveal_subsurface.py`):
- **Duration**: Change `duration_s` variable (line 47)
- **Frame rate**: Change `fps` variable (line 46)
- **Background color**: Change `BG_RGBA` variable (line 10)

After editing, rebuild:
```bash
python build.py
```

## 🎨 Styling Updates

All CSS is embedded in [src/index.html](../src/index.html) within the `<style>` tag.

To update styles:
1. Find the relevant CSS rules in the `<style>` section
2. Make your changes
3. Rebuild and test locally
4. Commit and push

Consider extracting CSS to a separate file (`src/css/style.css`) if styles grow complex.

## 🔍 Testing Changes

Always test locally before pushing:

1. **Build**:
   ```bash
   python build.py
   ```

2. **Serve locally**:
   ```bash
   python -m http.server --directory dist 8000
   ```

3. **Open in browser**: [http://localhost:8000](http://localhost:8000)

4. **Test**:
   - Check all pages load
   - Verify animations play
   - Test navigation links
   - Test form submission
   - Check responsive design (resize browser)
   - Test on mobile if possible

5. **If everything looks good**:
   ```bash
   git add .
   git commit -m "Describe your changes"
   git push origin main
   ```

## 📱 Adding a Blog

For future blog functionality:

1. Create a blog structure:
   ```
   src/
   ├── blog/
   │   ├── index.html       # Blog listing page
   │   ├── post-1.html      # Individual posts
   │   ├── post-2.html
   │   └── ...
   ```

2. Update `build.py` to copy the blog directory

3. Add a navigation link to the blog from the main page

4. Consider using a static site generator like [Jekyll](https://jekyllrb.com/) or [Eleventy](https://www.11ty.dev/) if blog grows large

## 🚨 Important Reminders

- ✅ **DO**: Edit files in `src/`
- ✅ **DO**: Run `python build.py` after changes
- ✅ **DO**: Test locally before pushing
- ✅ **DO**: Commit both `src/` changes and any script updates
- ❌ **DON'T**: Edit files in `dist/` (they will be overwritten)
- ❌ **DON'T**: Commit `dist/` to Git (it's in `.gitignore`)
- ❌ **DON'T**: Push without testing locally first
